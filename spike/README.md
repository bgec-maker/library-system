# Phase 0 — 카메라 · PWA 검증 스파이크

이 프로젝트의 **유일한 존재론적 위험**을 반나절에 확인합니다.

> **설치형 PWA(홈 화면 추가) 안에서 카메라가 켜지고 QR이 연속으로 읽히는가?**

❌ 라면 → PWA 설치를 포기하거나 Capacitor 래퍼를 검토해야 합니다. **React를 붙이기 전에** 알아야 합니다.

덤으로 **레포 + Pages + 배포 파이프라인**이 세워집니다. 어차피 필요한 것들입니다.

---

## 배포 (10분)

```bash
# 1. GitHub Organization → 새 레포 (public)
git init
git add .
git commit -m "Phase 0: 카메라·PWA 검증 스파이크"
git branch -M main
git remote add origin https://github.com/<ORG>/<REPO>.git
git push -u origin main
```

**2. Settings → Pages → Source: `Deploy from a branch` → `main` / `(root)` → Save**

몇 분 뒤 `https://<ORG>.github.io/<REPO>/` 에서 열립니다.

> HTTPS라야 카메라가 열립니다. Pages는 기본 HTTPS이므로 그대로 두면 됩니다.
> `.nojekyll` 파일이 들어 있습니다 — 지우지 마세요.

## 검사 (30분)

### ① 아이폰 — 이게 진짜 시험입니다

1. **Safari**로 위 주소를 엽니다 (Chrome 말고 Safari)
2. **공유 → 홈 화면에 추가**
3. **홈 화면의 아이콘으로 다시 엽니다** ← 이 단계를 건너뛰면 검사가 무의미합니다
4. `PWA로 설치되어 실행 중` 이 ✅ 인지 확인
5. **카메라 시작** → 아래 테스트 코드를 스캔

### ② 안드로이드

Chrome → 메뉴 → **앱 설치** → 설치된 앱으로 다시 열기 → 동일하게 확인

### ③ PC 웹캠

브라우저에서 그대로 열고 **카메라 시작**.
⚠️ **고정초점 저가 웹캠은 바코드를 못 읽습니다.** 실패하면 오토포커스 웹캠으로 재시도하세요.

### ④ 오프라인

카메라 확인 후 **비행기 모드** → 앱을 껐다 다시 열기 → 화면이 뜨면 Service Worker 정상.

## 이 스파이크가 미리 검증하는 것들

실제 앱에서 그대로 쓰게 될 것들입니다. 여기서 되면 나중에도 됩니다.

- **`BarcodeDetector` → ZXing 폴백** — iOS Safari는 `BarcodeDetector`가 없으므로 폴백이 반드시 필요합니다. 이 스파이크가 어느 쪽이 쓰이는지 알려줍니다.
- **ZXing을 로컬 번들로** — CDN에서 불러오면 오프라인에서 죽습니다.
- **스캔 피드백** — 성공 시 소리 + 초록 플래시 + 진동. 사서는 화면을 안 보고 작업합니다.
- **연타 방지** — 같은 코드가 초당 수십 번 읽히지 않도록 700ms 쿨다운.
- **후면 카메라** — `facingMode: environment`.

## ❌ 실패했다면

| 증상 | 의미 | 대응 |
|---|---|---|
| 브라우저 탭 ✅ / 설치형 ❌ | iOS PWA의 알려진 제약 | PWA 설치 포기(탭으로 사용) 또는 **Capacitor** 래퍼 |
| 둘 다 ❌ (`NotAllowedError`) | 권한 거부 | 설정 → Safari → 카메라 → 허용 |

| QR ✅ / ISBN ❌ | 초점 문제 | 오토포커스 웹캠, 또는 ISBN은 폰으로만 |
| 한 번만 읽히고 멈춤 | 디코더 루프 문제 | 결과 복사해서 공유 |
