# 33 · e2e-pipeline — 등록 파이프라인 회귀 스펙 CI 상주

## 왜
todo/28~32의 핵심 검증(BUSY_RETRY 자동 회복·큐 영속 재개)이 야간 로컬 하네스로만 수행되고
버려졌다 — 가짜 성공 금지 원칙의 연장: 검증은 반복 가능해야 한다. e2e.yml이 e2e/ 전체를
돌므로 스펙 파일 추가만으로 CI 상주가 된다.

## 무엇 (e2e/register-pipeline.spec.ts — smoke와 분리, 자체 목 라우팅)
1. BUSY_RETRY 2회 → 사용자 개입 없이 3차 자동 성공: 저장 직후 scan 화면 즉시 복귀 확인,
   트레이 재시도 상태 노출, 최종 등록번호 카드, 시도 횟수 정확히 3
2. 영속 재개: 1차 응답을 네트워크 실패로 끊고 → reload → 부팅 펌프가 같은 requestId로
   재전송해 완료(멱등 재개 경로)
3. 완료 트레이 지우기 동작
