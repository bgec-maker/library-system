# 115 · inventory-scan-e2e — 점검 세션 스캔 집계 무방비 + 목 바코드 Luhn 위반 (시각 감사 14R)

배경(캡처→프로브 실측): 점검 세션에서 목 카탈로그 바코드(2000001…)를 스캔 주입해도 0/3 —
원인은 제품이 아니라 **목 데이터**: 7자리 스캔은 Luhn 검증(scanBus, Code.gs 동일 가중치)을
거치는데 목 바코드가 전부 체크섬 위반이라 분류 단계에서 조용히 탈락. 그 결과 연 1회의 핵심
워크플로(점검 스캔→집계→분실 후보)가 **e2e 시야 밖**이었다.

할 일: 목 카탈로그 바코드를 Luhn 유효값(2000008/2000016/2000024/2000032)으로 교체(참조 스펙
동반 수정), inventory.spec 신설 — 시작(3권 대상)→유효 스캔 1권(1/3)→비대상(대출중) 스캔
무시→종료(분실 후보 2권) 상주 단정.

완료 조건: 신설 e2e 통과(14본), 전 게이트.

  산출) + 목에 근거 주석. 참조하는 다른 스펙 없음(전수 grep).
- e2e/inventory.spec 신설(14본째): 시작 3권 대상 → 유효 스캔 1/3 + inventoryScan 1회 →
  재스캔 멱등 → 대출중 스캔 무시 → 종료 분실 후보 2권(2000016·2000032, 스캔본 제외) 단정.
- 진단 기록: 캡처에서 0/3으로 보인 건 제품 결함이 아니라 목 체크섬 위반이 분류 단계에서
  탈락한 것 — 그 덕에 "핵심 워크플로가 e2e 시야 밖"이던 공백을 발견해 메웠다.
