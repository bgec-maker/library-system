# 50 · dock-redesign — 데스크톱 좌측 도크(사이드바) 재설계

## 현재 문제 (디자인 연구 후속 — 사용자 요청)
아이콘 10개가 라벨 없이 등간격 나열 — 신규 사용자(도우미·후임 사서)는 아이콘 의미를
호버 title 툴팁으로만 학습. 업무 화면과 관리 화면의 구분 없음. 열린 창 표시(is-open
brass)는 있으나 약함.

## 개선 (법전 준수: transform/opacity만, 새 hex 금지, 150~200ms, reduced-motion 무효)
1. 그룹 구분선: 업무(대출·등록·검색·점검) / 관리(대장·상세·최근·리포트·예약) / 시스템(설정)
   — registry 순서 유지, 시각 구분선만 추가
2. 호버 라벨 플라이아웃: 아이콘 우측으로 라벨이 나타남(opacity+transform, 포인터 있는
   기기 한정 @media (hover:hover)) — title 툴팁 지연 없이 즉시 학습 가능
3. 열린 창 표시 강화: is-open에 좌측 활성 바(brass) 추가 — 색약에도 위치 단서
4. 키보드 포커스 링 유지(:focus-visible brass) — 기존 규칙 그대로
## 완료 조건: 비포/애프터 스크린샷 + 전 게이트 + e2e(도크 셀렉터 불변 확인)
