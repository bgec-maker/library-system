# 54 · empty-pathways — 빈 상태에 다음 행동 버튼 (레퍼런스 점검 2-2)

근거: NN/g Empty States 가이드라인 3(빈 상태는 상태 설명 + 학습 단서 + **다음 행동 경로**).
우리는 1·2단계만 있었다.

- DataTable에 emptyAction?: {label, onClick} 프롭 신설 — 도메인 무지 원칙 유지(라벨은 호출측 t()).
- 적용 2곳: 최근 처리 → 「대출·반납 열기」, 예약 관리 → 「통합 검색에서 예약 걸기」.
- 미변상·검색 결과 없음 등 "행동 없음이 정상"인 목록엔 달지 않는다(프롭 주석에 명문화).
