# H1 · 모바일 카메라 무대 (긴급 — 등록 스프린트 품질)
증상: 카메라 동작 중인데 화면에 구분되는 UI가 없음 — 조준 불가·상태 불명.
## 스캔 활성 상태 (풀스크린 무대)
- 카메라 프리뷰(video) 전면 + **조준 프레임 = 실제 디코드 크롭 영역과 픽셀 일치** + 모서리 마크
- 상단 상태바: "스캔 중 → {수신 뷰 제목}" (scanBus 라우팅 대상 표시) · 유휴 자동종료 카운트다운(마지막 30초부터)
- 하단: 토치 토글(capabilities 지원 시만) · 연속 모드 핀 · 종료(44px)
- 인식 순간: 프레임 초록 플래시 + 인식 값 1초 표시 (기존 scanFeedback 연동 — 중복 구현 금지)
## 꺼짐 상태
- 큰 시작 버튼 카드 + "카메라는 필요할 때만 켜집니다" 한 줄 (ADR-020 카피)
## 제약
prefers-reduced-motion 준수 · 프리뷰 해상도 상한(저사양) 유지 · getUserMedia 호출점 1곳 불변 · 문자열 전부 t()
완료: 등록 뷰에서 켬→조준→인식 플래시→저장→자동 재개 흐름이 폰에서 명확 · 유휴 카운트다운 표시
