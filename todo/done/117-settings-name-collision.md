# 117 · settings-name-collision — 「설정」 이름 충돌 (시각 감사 15R)

배경: 같은 화면에 이름이 같은 두 목적지 — 도크/헤더의 기어(세션 연결: URL·토큰·작업자)와
레지스트리 설정 뷰(정책·무결성·스키마)가 둘 다 「설정」. 어느 걸 눌러야 하는지 이름만으로
구분 불가(고전적 IA 결함).

할 일: 기어 쪽을 「연결 설정」(en: Connection Settings)으로 — common.settings 리네임(소비처는
기어 2곳뿐 확인) + SessionGate 재열람 제목 동기화. 뷰 「설정」은 그대로.

완료 조건: 전 게이트, e2e(기존 통과 유지).
