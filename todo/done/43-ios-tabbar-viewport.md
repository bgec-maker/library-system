# 43 · ios-tabbar-viewport — 아이폰 하단 탭 배치 수정 (현장 제보)

## 증상 (사용자 아이폰 스크린샷, 2026-07-17)
하단 탭바가 화면 바닥에 붙지 않고 아래에 회색 죽은 띠가 떠 있음. 갤럭시는 정상.

## 원인
.m-shell { height: 100% } ← html/body/#root 100% 체인 — iOS Safari에서 100%는 "주소창이
보이는 작은 뷰포트" 높이로 고정된다. 스크롤로 주소창이 접히면 실제 화면은 커지는데 셸은
그대로라 그 차이가 탭바 아래 빈 띠로 노출. (탭바 자체의 safe-area-inset-bottom 처리는
이미 정상 — FRONTEND.md 플랫폼 주의의 "상단" 케이스와 짝인 "하단 동적 뷰포트" 케이스.)

## 수정
.m-shell에 @supports (height: 100dvh) { height: 100dvh } — 동적 뷰포트 추적(주소창 접힘
반영). 폴백은 기존 100%(iOS 15.3 이하·구형 = 현행 동작 유지). 설치형 PWA에선 dvh=100%라
무영향. 데스크톱 셸·student 표면은 증상 없음 — 범위 밖.

## 검증
CSS 단정 불가한 실기기 동작 — Pages 배포 후 사용자 아이폰 재확인(사파리 + 설치형 각 1회)을
완료 조건으로 한다. 로컬은 verify·e2e 회귀 없음만 확인.
