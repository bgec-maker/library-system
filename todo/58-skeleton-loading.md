# 58 · skeleton-loading — 첫 로딩 스켈레톤 ("대기업 감각" 3)

빈 화면+스피너는 "고장?"으로 읽히고, 스켈레톤은 "곧 온다"로 읽힌다(대중 앱 관례).
readCache 덕에 재방문은 즉시 뜨므로 대상은 **첫 로딩(캐시 0건)**뿐이다.

- DataTable: loading && rows 0건일 때 현행 "불러오는 중..." 텍스트 대신 스켈레톤 행
  (데스크톱 5행, 모바일 카드 3장). 펄스는 opacity 애니메이션만(backdrop-filter·그라디언트 시머 금지
  — 성능 예산). 토큰: --rule 바탕.
- 대시보드 베이스레이어 카드 3장: 첫 로딩 시 동일 스켈레톤.
- 스피너는 "행동 결과 대기"(등록 저장 중·발급 중)에만 남긴다 — 역할 분리를 주석으로 명문화.
- aria: 스켈레톤 컨테이너 aria-busy="true", 텍스트 t('common.loading') sr-only 유지.

## 완료 조건: 첫 로딩 스켈레톤 컷(카탈로그·대시보드) · 캐시 재방문 시 스켈레톤 미출현 확인 · 전 게이트·e2e
