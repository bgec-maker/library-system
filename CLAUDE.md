## 🔴 절대 규칙 (개정 반영판)

1. **GAS 안에서 카메라 불가** (공식 정책) → 스캔·웹앱 프론트는 반드시 GAS 밖(Pages). 사이드바는 타이핑 전용.
2. **`executeWrite_`·보호 로직·예약 배정·감사 로그는 수정 금지** — 검증된 자산. 패치는 그 주변에만.
3. **ISBN은 관리 ID가 아님** (ADR-003 — 복본 문제). barcode = 6자리+Luhn (P5).
4. **책 QR = URL / 학생 QR = 불투명 ID.** 도메인 확정 전 라벨 인쇄 금지.
5. **셀프 대출 O / 셀프 반납 X** (웹앱 단계, ADR-010).
6. **행 삭제로 '삭제' 구현 금지** — 상태 코드 변경. (아카이브 이동의 행 삭제만 예외 — P2-b)
7. iOS: `BarcodeDetector` 없음 → ZXing 로컬 번들 + Worker + 크롭 + 10fps 스로틀.
8. 파생 뷰를 O(n²) 수식으로 만들지 말 것 (v3에서 실증한 함정 — DECISIONS 방법론 절).
9. 스키마 변경은 실데이터 투입 전엔 재생성, 투입 후엔 마이그레이션. HEADERS 상수와 시트 동기화 필수.

## 검증 원칙 (이 프로젝트에서 실증됨)

- **문서보다 더미 데이터** — 시트 3개 버전에서 버그 9개를 실물로 잡았다 (연체 은폐, 재대출 증발, 설정 행 밀림, O(n²)…).
- **수식이 계산된다 ≠ 답이 맞다.** 오류 0개 워크북이 조용히 틀린 답을 낸다. 값을 직접 검증하라.
- 실수를 일부러 심어 시스템이 잡는지 확인하라.
- 패치마다 PATCH_SPEC의 수용 기준을 실데이터 규모(5,000권·10,000대출 모의)로 통과시켜라.

## 🟡 school-patch-v1 검증 노트

- 제작 환경에 GAS 런타임이 없어 **코드 실행 미검증** (구문·값 검증만). PATCH_NOTES의 9단계 프로토콜을 실제 시트에서 수행 후 승격.
- 불일치 1건: PATCH_NOTES가 "Code.gs.orig 보존" 주장하나 **패키지에 .orig 없음** → 원본은 `mvp-package/`가 그 역할 (diff 기준으로 사용).
- ADR-007 주의: v1은 번호를 **텍스트 서식(@)** 으로 구현 (ADR 원문은 "값은 숫자+표시 서식"). Luhn 검증숫자가 붙어 산술 불필요하므로 수용 가능하나, **수기 입력 시 자동 숫자변환으로 0이 깎이는 케이스**를 검증 항목에 추가할 것.
