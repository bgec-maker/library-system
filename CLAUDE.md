## 🔴 절대 어기면 안 되는 것 (전문은 `docs/HANDOFF.md` §2)

대부분 **에러 없이 조용히 실패**합니다. 그래서 위험합니다.

1. **GAS 안에서 카메라는 작동하지 않습니다.** Google 공식 정책(2026-06). 프론트는 반드시 GAS 밖에.
2. **스프레드시트를 정렬하면 시스템이 파괴됩니다.** `도서ID = 행 번호 − 1`. 정렬 한 번에 라벨 5,000장이 무효화됩니다.
3. **행 삭제 금지.** 폐기·졸업은 `상태` 변경. **ID 재사용 금지.**
4. **책의 속성을 ID에 인코딩 금지** (난이도·분류 구간 등). 속성은 바뀌고 ID는 불변.
5. **`opId` 멱등키 없으면 대출이 중복 생성됩니다.**
6. **카메라는 싱글턴.** 두 컴포넌트가 `getUserMedia` → 충돌. **`<video>` 언마운트 금지.**
7. **권한 검증은 서버에서.** UI 숨김은 보안이 아님. Pages 레포는 공개이므로 **프론트에 비밀값 금지.**
8. **셀프 대출 O / 셀프 반납 X** (직관과 반대. 이유는 HANDOFF §2-⑦)
9. **도메인 확정 전 라벨 인쇄 금지.** QR에 URL이 박힙니다.
10. **iOS Safari: `BarcodeDetector` 미지원** → ZXing 폴백 필수. **`toBlob('image/webp')`가 조용히 PNG로 폴백**함 → 타입 확인 후 JPEG 폴백.

## 코드 규칙 (전문은 `docs/HANDOFF.md` 코드 품질 절)

**단일 통로를 만드세요.** 각각 "여기 하나만 고치면 되는" 지점입니다.

- **셸 / 뷰 분리** — 뷰에 타이틀바·닫기 버튼 넣지 말 것. 데스크톱↔모바일 재사용의 전제
- **`api.call(action, payload, opId)`** — 모든 서버 통신이 이 함수 하나를 통과. 토큰 부착도 여기서만
  → 로그인 방식이 아직 미정이므로 **특히 중요합니다**
- **`CameraService` 싱글턴** — 카메라를 만지는 코드는 여기뿐. 윈도우 시스템 **바깥**에 삽니다
- **시트 접근 레이어** — `getBook(id)` 같은 함수로 감쌀 것. `getRange()`를 여기저기 뿌리지 말 것
- **공유 타입** — `Book` `Student` `Loan`을 프론트/백이 같은 파일에서 import (TypeScript)
- **상태값 상수화** — 코드는 `BOOK_STATUS.LOANED`, 시트는 `대출중`. 매핑 한 곳
- **에러 코드** — `{ok:false, code:'OVERDUE_BLOCKED'}`. 메시지 문자열로 분기 금지
- **`Config` 시트** — 모든 상수. 코드에 박지 말 것

**중복 3번 규칙:** 두 번까진 복사, **세 번째에 추출.** 성급한 추상화가 중복보다 나쁩니다.
