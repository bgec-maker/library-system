# FEATURES.md — 작은 학교를 위한 기능 지도

> 설계 명제: **작은 학교의 무기는 통계가 아니라 개별성이다.**
> 전교 수백 명 규모에선 전수 케어가 가능하다. 모든 기능은 집계 수치가 아니라
> **이름과 행동**(누구에게 · 무엇을 하면 되는지)으로 끝나야 한다.
>
> 진입점: 데스크톱 대시보드의 「조용한 신호」 패널 (FRONTEND.md 대시보드 절).
> 우선순위 근거: **R1은 로그인 불필요** — 사서 세션만으로 동작. 로그인 미결(🟡)을 우회한다.

> **구현 현황 (2026-07-18)**: R1 1~5 + R3 10 **전부 가동 중** — `reports` 뷰(허브)에서 미리보기·인쇄,
> 진입은 대시보드 「조용한 신호」. 여기에 더해 **인사이트 시각화 12종**(대출 잔디·분류 트리맵·회전율
> 사분면·예약 압력·시간대 파도·연체 흐름·학급 참여 링·12개월 곡선·서가 온도·장서 연령·학년 격차·
> 예산 그림)이 대시보드 기저층에 상주한다(`VIZ.md`). R2(6~9)는 🟡 로그인 확정 대기 그대로.

## R1 · 사서 리포트 (로그인 불필요 — 데이터 이미 존재. 최우선)

| # | 기능 | 데이터 소스 | 출력 |
|---|---|---|---|
| 1 | **미대출 학생 발굴** — 기간 내 대출 0회 재학생 명단 | MEMBERS − LOANS | 반별 이름 목록. "숫자"가 아니라 "명단" |
| 2 | **담임 리포트** (월간·반별) — 대출 현황, 미대출 명단, 연체, 우리 반 인기책 | LOANS+MEMBERS+TITLES | **인쇄 전제** A4 1장. 소형교는 메신저보다 종이가 현실적 |
| 3 | **죽은 장서 / 구매 추천** — 입수 2년↑·대출 0회 = 폐기 후보 / 예약 누적·회전율 상위 = 복본 구매 후보 | COPIES(대출횟수·입수일)+RESERVATIONS | 예산이 적을수록 "뭘 살지"가 데이터여야 함 |
| 4 | **회수 쪽지** — 연체·방학 미반납을 담임별로 쪽지 인쇄 | LOANS 미반납+MEMBERS 학급 | 교실 전달용 절취 쪽지 |
| 5 | **기증 감사장** — 연말 기증자별 목록·감사장 일괄 생성 | COPIES.기증자·단가 | 기증 선순환 유도 |

구현 형태: 레지스트리에 **`reports` 뷰 1개** (리포트 허브 — 종류 선택 → 미리보기 → 인쇄). 뷰 비대화 방지.
백엔드: doPost 읽기 액션 `report`(type 파라미터) + `dashboard` — 기존 `readTable_`/`getDashboardData_` 재사용, 쓰기 없음.

## R2 · 학생 대면 (🟡 로그인 결정 후)

| # | 기능 | 비고 |
|---|---|---|
| 6 | **반 대항 함께 읽기** | 소형교에서 전교 랭킹은 금방 고정됨(같은 아이가 1등) → **반 단위 협력 지표**(우리 반이 함께 읽은 페이지)가 참여를 만든다. 페이지 가중은 기존 결정 그대로 |
| 7 | **첫 대출 축하** | 학생의 첫 LOANS 감지 → 화면 축하 + 인쇄 카드. 미대출 발굴(R1-1)의 짝 기능 |
| 8 | **"이 책 다음엔"** | LOANS 동시대출 연관 → 희소하면 알라딘 카테고리·난이도·권장학년 폴백 |
| 9 | **독서 이력 리포트** | 독서기록(사서 확인분) → 학생별 출력. 생기부 참고용 증빙 |

## 원칙

- 리포트는 전부 **읽기 전용 백엔드 액션** — 보호·멱등 체계와 무관하게 안전
- 개인정보: 리포트는 사서 화면·인쇄물에만. 학생 대면 표면에 **다른 학생의 미대출·연체를 절대 노출하지 않는다**
- "조용한 신호"의 각 줄은 해당 리포트로 **직행하는 버튼** — 대시보드는 보는 곳이 아니라 시작하는 곳
