# DESIGN.md — 디자인 시스템 (모든 트랙의 횡단 규율)

> Claude Code는 UI를 만들거나 고치기 전에 이 문서를 따른다. 여기 없는 스타일 결정을 새로 내리면 이 문서에 추가부터 한다 — **문서에 없는 임의 hex·px·폰트 금지.**

## 두 표면, 한 시스템

| | work (사서·스테이션) | student (학생 공개) |
|---|---|---|
| 성격 | 도서관의 종이와 목재 — 차분·밀도·속도 | 발견의 놀이 — 표지가 주인공 |
| 팔레트 | paper `#f2f0ea` · deep navy `#1b3a4b` · brass `#d9a441` (스파이크 계승) | 보라 `#6d3fd6` · 오렌지 `#ff8a3d` 계열 |
| radius | 6px (각지게) | 14px (둥글게) |
| 위계 주인공 | 숫자·이름 (업무 대상) | 책 표지·별점 |

**같은 변수명, 다른 값** — 컴포넌트는 `var(--token)`만 참조. 스킨 파일 교체가 곧 표면 전환. 색상 리터럴은 tokens/ 두 파일에만 존재한다.

## 토큰 (현행 + 이번에 추가)

- 색: `--ink/-2/-3 · --paper · --panel · --rule · --deep · --brass · --pass/--fail/--wait/--idle` (기존)
- 타입: `--fs-xs(12)~--fs-2xl(26)` 6단 (기존) — **12px 미만 금지**, 등록번호·바코드는 `--mono` + 크게(clamp)
- **간격(추가)**: `--sp-1(4) --sp-2(8) --sp-3(12) --sp-4(16) --sp-6(24) --sp-8(32)` — 마진·패딩 px 리터럴 치환
- **시각화 램프(추가, VIZ 전용)**:
  - 순차(히트맵): `--viz-seq-1~5` = paper→deep 5단 보간
  - 발산(±비교): fail↔paper↔pass
  - 범주(≤6): deep · brass · pass · wait · ink-2 · fail — **이 순서 고정** (차트마다 색 의미가 흔들리지 않게)

## 컴포넌트 상태 (전 표면 공통)

- 포커스: `:focus-visible` 2px `var(--brass)` outline (기존) — 커스텀 컴포넌트도 예외 없음
- busy: 버튼 스피너 대신 라벨 교체("저장 중…") + disabled — 레이아웃 점프 금지
- 비어 있음: 회색 아이콘 + **다음 행동 한 줄** ("아직 기록이 없어요 — 첫 책을 스캔해보세요"). 빈 화면은 초대장이다
- 오류: 무엇이 잘못됐고 무엇을 하면 되는지. 사과·모호함 금지. 서버 코드는 접힌 상세로
- 터치 타깃 ≥44px(모바일) — 실행취소 버튼 포함

## 인쇄 (리포트 R1·R3의 절반은 종이다)

- `@media print`: 잉크 절약 — 배경 제거·테두리 유지, deep→검정, A4 세로, 리포트당 1장 원칙
- 회수 쪽지: 절취선(dashed) + 한 반이 한 열
- 모든 인쇄물 머리: 학교명 · 생성일 · "○○학교 도서관 시스템" — 꼬리: 페이지 번호

## 테이블 (공용 DataTable 규칙)

- 행높이 40px · 구분선은 헤어라인 `--rule` · **zebra 금지**(종이 위 괘선 느낌) · hover는 paper 톤
- 정렬 화살표 `--brass`, 활성 정렬 열 헤더는 `--deep`
- 숫자 열 우측 정렬, 등록번호는 `--mono`
- 인쇄 시 페이지네이션 해제 후 전체 인쇄 — 단 200행 초과면 확인 다이얼로그

## 아이콘·모션·카피

- 아이콘: lucide-react만. 탭 20 / 도크 24 / 타이틀바 14 / 본문 인라인 16. 이모지는 student 표면 콘텐츠(체감난이도 😊😐😰)에만
- 모션: 150~200ms ease-out, `prefers-reduced-motion`이면 전부 무효(기존). 장식 애니메이션 금지 — 스캔 플래시·시트 슬라이드만
- 카피: 사용자 언어(시스템 내부어 금지 — "ScriptProperties" 사건 재발 방지) · 능동태 · 버튼=일어날 일 그대로("대출 취소", "쪽지 인쇄") · 같은 행동 같은 이름 관통

## 스킬화 여부 (사용자 질문의 답)

외부에서 구해올 스킬은 없다 — 범용 디자인 스킬은 이 프로젝트의 고유 결정(두 표면·램프·인쇄)을 모른다. **이 문서가 곧 프로젝트 전용 디자인 스킬의 원고**다. 운용 규칙: Claude Code가 이 문서를 두 번 이상 어기면, 그때 skill-creator로 `library-design` 스킬로 승격해 자동 트리거시킨다. 그 전엔 문서로 충분 — 과설계 방지.
