# VIZ.md — 시각화 카탈로그

> 원칙 셋:
> ① **집계는 서버 일배치.** 일일 트리거가 LOANS(+아카이브)를 훑어 `VIZ_CACHE` 시트에 요약 적재 → doPost `viz` 액션은 읽기만. 클라이언트가 원장을 스캔하지 않는다(성능·할당량).
> ② **차트 라이브러리 없이 수제 SVG 우선.** 히트맵·트리맵·바·라인은 직접 그린다 — 토큰 색 직결, 번들 절약(학생 번들 보호), 인쇄 친화. 복잡도가 정당화될 때만 라이브러리 도입을 별도 결정.
> ③ **모든 시각화는 행동으로 끝난다.** 그림 옆엔 항상 "그래서 뭘 하나" 버튼(리포트·목록 직행). 개인 식별 정보는 사서 표면에서만.
