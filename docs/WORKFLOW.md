## 원칙: 가장 불확실한 것을 가장 먼저

기능 순서대로 만들면, **가장 위험한 걸 가장 나중에** 만나게 됩니다. 그때는 이미 매몰비용이 큽니다.
이 프로젝트의 최대 위험은 **"아이폰 PWA에서 카메라가 켜지는가"** 입니다. 그러니 그것부터 합니다.

## Phase 0 — 위험 검증 스파이크 ✅ **완료 (2026-07-13)**

**🟢 통과.** iPhone 설치형 PWA에서 카메라 + QR/EAN-13 연속 스캔 48건 성공.
**GitHub Pages + React + PWA 아키텍처가 유효합니다.** (상세: `HANDOFF.md` §2-⑨)

남은 확인: ✈️ 오프라인(비행기 모드) · 🤖 안드로이드 · 💻 **PC 웹캠(고정초점 웹캠은 EAN-13 실패)**

- 0.1 GitHub 계정/Organization + 빈 레포
- 0.2 정적 HTML **한 장** — manifest + Service Worker + 카메라 + QR 디코더
- 0.3 GitHub Pages 배포
- 0.4 **아이폰: 홈 화면에 추가 → 설치된 PWA에서 카메라 → QR 읽기**
- 0.5 안드로이드 · PC 웹캠에서도 동일 확인

🚩 **관문:** 세 기기 모두에서 QR이 읽힌다.

> 설치형 iOS PWA에서 `getUserMedia`가 과거에 자주 깨졌습니다. 지금은 대체로 되지만 **실제 기기에서 확인 전엔 확정이 아닙니다.**
> ❌라면 → 네이티브 래퍼(Capacitor) 또는 "기본 카메라 + URL QR" 방식 전면 채택으로 **아키텍처를 갈아엎어야 합니다.** 반나절로 이걸 알아내는 게 3개월 뒤에 아는 것보다 낫습니다.

## Phase 1 — 기반 준비 (리드타임 있음. Phase 2와 병렬 가능)

- 1.1 **프로젝트 전용 Gmail 계정** 생성 — 기관 소유, 개인 계정 금지
- 1.2 **도메인 구매 + DNS** ← 라벨 인쇄 전 필수. 되돌릴 수 없음
- 1.3 GitHub Organization + 레포 정식 구성 (브랜치·Actions·시크릿)
- 1.4 **Google Sheets 부트스트랩** — 시트 13종 + 헤더 + 데이터 검증 + 보호 + README 탭
- 1.5 **등록번호 체계 확정** — 6자리 연번 + Luhn 체크문자, 라벨 규격, QR 최소 15mm
- 1.6 시드 데이터 CSV 임포트 (Phase 5 테스트용)
- 1.7 서점 **납품 명세서(ISBN·서명·저자·페이지수, 가능하면 MARC)** 확보 시도 → 0단계 절반 절약

## Phase 3 — 스캐너 & 셸 (가장 어려운 부분) 🚩

- 3.1 **CameraService 싱글턴** — 스트림은 하나만

🚩 **관문:** 아이폰 · 안드로이드 · PC 웹캠에서 QR이 **연속으로** 읽히고, 창을 옮겨도 스캔이 끊기지 않는다.
